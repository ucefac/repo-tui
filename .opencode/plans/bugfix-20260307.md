## Bug 汇总

### Bug 1: 主界面工具栏没了

**现象**: 主界面底部的工具栏快捷键提示消失或被路径信息覆盖

**根因分析**:

当前布局（状态栏高度 = 2 行）：
```
┌─────────────────────────────┐  ← 第 0 行 (上边框)
│ [j/k]Nav [g/G]Jump...       │  ← 第 1 行 (内容/路径)
└─────────────────────────────┘  ← 第 2 行 (下边框，被路径覆盖)
```

**问题**:
1. `StatusBar` 使用 `Borders::ALL`，需要 3 行空间（上边框 + 内容 + 下边框）
2. 但实际只分配了 2 行（`render.rs:122`）
3. 路径信息渲染在最后一行（`status_bar.rs:97`），覆盖了下边框和内容

**修复方案**: 修改 `StatusBar` 渲染逻辑，在 Block 内部正确分隔状态消息和路径信息

**修改文件 1**: `src/ui/render.rs:122`
```rust
// 修改前
Constraint::Length(2), // Status bar (reduced from 4 to 2)

// 修改后
Constraint::Length(4), // Status bar (1 border + 1 status + 1 path + 1 border)
```

**修改文件 2**: `src/ui/widgets/status_bar.rs:57-100`

完整替换 `impl<'a> Widget for StatusBar<'a>`:

```rust
impl<'a> Widget for StatusBar<'a> {
    fn render(self, area: Rect, buf: &mut Buffer) {
        // Create block with border
        let block = Block::default()
            .borders(Borders::ALL)
            .border_style(Style::default().fg(self.theme.border_normal))
            .style(Style::default().bg(Color::DarkGray));

        // Calculate inner area (excluding borders)
        let inner = block.inner(area);
        
        // Render block first
        block.render(area, buf);

        // Check minimum inner height
        let inner_height = inner.height as i16;
        if inner_height < 1 {
            return;
        }

        // Status message
        let status_text = if self.loading {
            format!("⏳ {}", self.status_message)
        } else if self.error {
            format!("⚠️ {}", self.status_message)
        } else {
            self.status_message.to_string()
        };

        // Render status and path in inner area
        if inner_height >= 2 {
            // Status on top row
            let status_area = Rect::new(inner.x, inner.y, inner.width, inner_height - 1);
            let status_paragraph = Paragraph::new(status_text)
                .style(Style::default().fg(self.theme.text_secondary))
                .wrap(Wrap { trim: true })
                .alignment(Alignment::Left);
            status_paragraph.render(status_area, buf);

            // Path on bottom row (if available)
            if let Some(path) = self.path {
                let path_text = format!(
                    "📂 {}{}",
                    path.display(),
                    self.repo_count
                        .map(|c| format!(" ({} repos)", c))
                        .unwrap_or_default()
                );

                let path_paragraph = Paragraph::new(path_text)
                    .style(Style::default().fg(self.theme.text_secondary))
                    .alignment(Alignment::Left);

                let path_area = Rect::new(inner.x, inner.y + inner_height - 1, inner.width, 1);
                path_paragraph.render(path_area, buf);
            }
        } else {
            // Only 1 row, render status only
            let status_paragraph = Paragraph::new(status_text)
                .style(Style::default().fg(self.theme.text_secondary))
                .wrap(Wrap { trim: true })
                .alignment(Alignment::Left);
            status_paragraph.render(inner, buf);
        }
    }
}
```

### Bug 2: 搜索激活时，m 没有被禁用

**现象**: 搜索激活时按 m 键，直接打开目录选择器，搜索框没有收到'm'输入

**根因**: 
消息队列延迟处理导致状态不一致：

1. 用户快速连续按 `/` 然后按 `m`
2. `/` 键处理：立即设置 `app.search_active = true`（直接修改状态）
3. `m` 键处理：发送 `AppMsg::ShowDirectoryChooser` 到消息队列
4. 消息循环处理：从队列中取出 `ShowDirectoryChooser` 消息并处理
5. 打开目录选择器（**忽略了当前 search_active 状态**）

**关键问题**: `AppMsg::ShowDirectoryChooser` 的消息处理没有检查 `search_active` 状态

**修复方案**: 在 `AppMsg::ShowDirectoryChooser` 处理中添加搜索状态检查

**修改文件**: `src/app/update.rs:177-187`

**修改内容**:
```rust
// 在 AppMsg::ShowDirectoryChooser => { 之后立即添加检查
AppMsg::ShowDirectoryChooser => {
    // 如果搜索激活，忽略此消息（防止延迟消息错误打开选择器）
    if app.search_active {
        return;
    }
    
    app.state = AppState::ChoosingDir {
        path: dirs::home_dir().unwrap_or_default(),
        entries: Vec::new(),
        selected_index: 0,
        scroll_offset: 0,
    };
    runtime.dispatch(crate::app::msg::Cmd::ScanDirectory(
        dirs::home_dir().unwrap_or_default(),
    ));
}
```
