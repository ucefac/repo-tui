## 文档规范

### 创建规则

1. **文件名格式**: `<brief-description>.md`（简短描述，小写，`-` 连接）
2. **Bug ID 格式**: `[模块]-[序号]`，如 CONFIG-001, UI-002
3. **必须包含**: Bug ID、严重程度、根因分析、修复方案、测试用例

### 文档模板

```markdown
# [Bug 标题]

**Bug ID**: XXX-XXX
**严重程度**: 🔴 高 / 🟡 中 / 🟢 低
**状态**: 待修复 / 修复中 / 已修复
**创建时间**: YYYY-MM-DD

## 问题摘要
## 根因分析
## 修复方案
## 测试验证
## 影响评估
```

### 严重程度定义

| 级别 | 图标 | 说明 | 响应时间 |
|------|------|------|----------|
| 高 | 🔴 | 系统崩溃、数据丢失、安全漏洞 | 立即 |
| 中 | 🟡 | 功能异常、性能下降 | 24 小时内 |
| 低 | 🟢 | 界面问题、轻微不便 | 下次迭代 |
