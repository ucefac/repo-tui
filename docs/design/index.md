## 设计规范

### 核心原则

| 原则 | 说明 |
|------|------|
| **简洁** | 高度仅 1 行，不影响主要内容显示 |
| **清晰** | 标题文字清晰可读，使用主题色 |
| **一致** | 与现有 UI 风格保持一致 |
| **主题化** | 不同主题有不同的标题颜色效果 |

### 创建规则

1. **文件名格式**: 小写，单词间用 `-` 连接，如 `component-name.md`
2. **必须包含**: 设计概述、UI 规范、实现细节
3. **与代码同步**: 文档与代码实现保持同步更新

### 文档模板

```markdown
# [组件/系统名称] 设计文档

**文档版本**: x.x
**创建日期**: YYYY-MM-DD
**状态**: 待实施/已实施

## 设计概述
## UI 设计
## 功能规格
## 技术实现
## 测试验证
```
