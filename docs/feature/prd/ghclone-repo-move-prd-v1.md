# 仓库移动功能产品需求文档

**版本**: v1
**创建时间**: 2026-03-13
**作者**: Product Manager

---

## 1. 概述

### 1.1 项目背景

repotui 是一个 GitHub 仓库管理 TUI 工具，用户可以管理多个主目录下的仓库。在实际使用中，用户经常需要将仓库从一个主目录重新组织到另一个主目录下。目前，这个操作需要用户手动在文件系统中完成，包括：
1. 退出 TUI
2. 使用命令行或文件管理器移动目录
3. 重新启动 TUI 刷新仓库列表

这个流程繁琐且打断了用户的工作流。

### 1.2 目标用户

- 需要在多个主目录之间重新组织仓库的开发者
- 使用 TUI 进行日常仓库管理的用户
- 希望通过快捷键快速完成仓库管理的用户

### 1.3 业务目标

1. **提升用户体验**：无需退出 TUI 即可完成仓库移动
2. **提高效率**：通过快捷键快速触发移动操作
3. **保持数据一致性**：移动后自动刷新仓库列表，保持状态同步

## 2. 功能需求

### 2.1 功能列表

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 按键触发移动 | 用户通过按键触发仓库移动操作 |
| P0 | 主目录选择 | 用户可以选择目标主目录 |
| P0 | 智能跳过 | 如果目标目录相同则跳过 |
| P0 | Toast 提示 | 移动成功/失败显示提示消息 |
| P1 | 冲突处理 | 目标位置已有同名仓库时提示 |

### 2.2 详细功能描述

#### 2.2.1 按键触发移动

- **触发按键**: `M`（Shift+m，避免与现有的 `m` 主目录管理冲突）
- **触发条件**:
  - 在仓库列表界面（Running 状态）
  - 有选中的仓库
  - 配置中至少有两个主目录（否则无意义）
- **交互流程**:
  1. 用户按下 `M` 键
  2. 弹出主目录选择器（模态对话框）
  3. 用户选择目标主目录
  4. 确认后执行移动操作

#### 2.2.2 主目录选择器

- **主目录来源**: 从配置文件中读取的已启用的 `main_dirs` 列表
- **选择器样式**: 弹出式下拉菜单/列表
- **排除项**: 当前仓库所在的主目录（如果可识别）应标记但可选
- **快捷键**:
  - `↑/↓`: 导航
  - `Enter`: 确认选择
  - `Esc`: 取消

#### 2.2.3 智能跳过逻辑

- **检测逻辑**: 比较目标主目录路径与当前仓库所在主目录路径
- **跳过行为**: 如果相同，不执行移动，显示提示信息"仓库已在该主目录中"
- **Toast 类型**: Info（非成功，非错误）

#### 2.2.4 Toast 提示系统

需要新增 Toast 消息系统：

| 场景 | 消息内容 | 类型 | 时长 |
|------|----------|------|------|
| 移动成功 | "{{仓库名称}} 移动到{{主目录名称}}成功" | Success | 3 秒 |
| 已跳过 | "{{仓库名称}}已在{{主目录名称}}中" | Info | 2 秒 |
| 移动失败 | "移动失败：{{错误原因}}" | Error | 5 秒 |
| 冲突警告 | "{{目标位置}}已存在同名仓库" | Warning | 5 秒 |

#### 2.2.5 仓库类型支持

- **standard 仓库**: 普通目录，直接移动整个目录
- **git 仓库**: 包含.git 目录的仓库，移动后保持 git 状态不变

### 2.3 移动操作流程

```
1. 用户按下 M 键
2. 检查是否有选中的仓库
   - 无 → 显示提示"请先选择一个仓库"
3. 检查配置中是否有至少 2 个主目录
   - 否 → 显示提示"需要至少 2 个主目录"
4. 弹出主目录选择器
5. 用户选择目标主目录
6. 检查目标目录是否与当前相同
   - 是 → 显示 Info Toast，跳过
7. 执行移动操作（后台）
   - 检查目标位置是否已有同名仓库
     - 是 → 显示冲突对话框（确认覆盖/重命名/取消）
   - 执行文件系统移动
8. 移动完成
   - 成功 → 刷新仓库列表，显示 Success Toast
   - 失败 → 显示 Error Toast
```

## 3. 非功能需求

### 3.1 性能要求

- **移动操作响应时间**:
  - 小仓库 (<100MB): < 2 秒
  - 中等仓库 (100MB-1GB): < 10 秒
  - 大仓库 (>1GB): 显示进度条
- **UI 不阻塞**: 使用异步操作，移动过程中保持 UI 响应

### 3.2 安全要求

- **路径验证**: 确保目标路径在允许的主目录范围内
- **权限检查**: 确保有读取源目录和写入目标目录的权限
- **符号链接处理**: 不跟随符号链接，直接移动符号链接本身
- **原子性**: 移动操作应尽可能原子，避免中途失败导致数据丢失

### 3.3 兼容性要求

- **操作系统**: macOS/Linux/Windows
- **路径格式**: 支持包含空格的仓库名和路径
- **文件名编码**: 支持 UTF-8 路径

### 3.4 可靠性要求

- **错误恢复**: 移动失败时，源仓库应保持原状
- **状态同步**: 移动成功后，仓库列表自动刷新
- **数据完整性**: 确保移动后文件内容完整，git 状态不变

## 4. 验收标准

### 4.1 功能验收

| 编号 | 测试项 | 预期结果 |
|------|--------|----------|
| F1 | 按下 M 键能够触发移动操作 | 弹出主目录选择器 |
| F2 | 无选中仓库时按 M 键 | 显示提示"请先选择一个仓库" |
| F3 | 只有 1 个主目录时按 M 键 | 显示提示"需要至少 2 个主目录" |
| F4 | 主目录选择器正确显示 | 显示所有已启用的主目录 |
| F5 | 选择相同主目录 | 显示 Info Toast，跳过移动 |
| F6 | 移动成功后显示正确的 Toast 提示 | 显示"{{仓库名称}}移动到{{主目录名称}}成功" |
| F7 | standard 仓库可以正常移动 | 移动后文件完整 |
| F8 | git 仓库可以正常移动 | 移动后.git 状态不变 |
| F9 | 移动后仓库列表自动刷新 | 原位置仓库消失，新位置出现 |
| F10 | Esc 键取消移动操作 | 关闭选择器，不执行移动 |

### 4.2 异常场景

| 编号 | 测试项 | 预期结果 |
|------|--------|----------|
| E1 | 目标位置已有同名仓库 | 显示冲突对话框，提供覆盖/重命名/取消选项 |
| E2 | 无写入权限 | 显示 Error Toast，源仓库保持原状 |
| E3 | 源目录被其他进程占用 | 显示 Error Toast，操作失败 |
| E4 | 目标路径不存在 | 自动创建目标路径或显示错误 |
| E5 | 移动过程中取消 | 如已开始移动，无法取消，等待完成 |

### 4.3 性能验收

| 编号 | 测试项 | 预期结果 |
|------|--------|----------|
| P1 | 小仓库移动 (<100MB) | < 2 秒完成 |
| P2 | 移动过程中 UI 响应 | 可以继续导航、搜索等操作 |

## 5. 技术实现建议

### 5.1 核心模块

1. **keyboard.rs**: 添加 `M` 键处理
2. **msg.rs**: 添加新的消息类型 `MoveRepository`
3. **update.rs**: 添加移动仓库的更新逻辑
4. **ui/render.rs**: 添加主目录选择器渲染
5. **repo/move.rs**: 新增模块，处理仓库移动逻辑
6. **toast.rs**: 新增模块，处理 Toast 消息

### 5.2 数据结构变更

```rust
// msg.rs
pub enum AppMsg {
    // ... 现有消息 ...
    /// 开始移动仓库
    MoveRepository,
    /// 选择目标主目录
    MoveTargetSelected(usize),
    /// 取消移动
    CancelMove,
    /// 移动完成
    MoveCompleted(Result<(), MoveError>),
}

// AppState 新增状态
pub enum AppState {
    // ... 现有状态 ...
    /// 选择移动目标目录
    ChoosingMoveTarget {
        /// 目标主目录列表
        targets: Vec<PathBuf>,
        /// 选中索引
        selected_index: usize,
    },
}
```

## 6. 风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 移动大仓库时性能差 | 中 | 中 | 显示进度条，异步执行 |
| 移动过程中断电 | 低 | 高 | 无法完全避免，但保持源文件直到完成 |
| 权限问题 | 中 | 中 | 提前检查权限，明确错误提示 |
| 符号链接处理 | 低 | 中 | 文档说明，不跟随符号链接 |
